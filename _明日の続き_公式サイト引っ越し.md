# 明日やること: 公式サイトを親ドメイン(sekkotsuin-komaki.com)へ引っ越し

## いまの状態（2026-07-21時点）

- **親ドメイン https://sekkotsuin-komaki.com/** … 旧・公式サイト(WordPress)が稼働中
  - タイトル「BTG接骨院 小牧院 | 小牧市で骨盤矯正、交通事故のむち打ち…」
  - XServer(btgkomaki)のWordPress簡単インストール(id=2eb7a0a47f8b7f8b)。DBあり
- **新・公式サイト（リメイク版）** … https://bonetechgroup.github.io/btg-komaki-site/ で仮公開中
  - リポジトリ: github.com/BoneTechGroup/btg-komaki-site
- 事故サイト側の「公式サイト」リンク(全34ページ+フッター)は現在github.ioを向いている

## ゴール

sekkotsuin-komaki.com を開くと新公式サイトが表示される。旧WPの評価とGoogleマップからのリンクを自前ドメインで引き継ぐ。

## ⚠️ 最重要注意（消してはいけないもの）

親ドメインのドキュメントルート `/sekkotsuin-komaki.com/public_html/` の中には
**事故サイトの実体フォルダ `jiko.sekkotsuin-komaki.com/`（中に /blog/ のWordPressも）が同居している。**
デプロイや削除でこのフォルダを絶対に巻き込まないこと（FTP-Deploy-Actionのexclude必須）。

## 手順

1. **旧WPのバックアップ**
   - ファイル: FTPで public_html直下の旧WP一式をローカルに退避（wp-config.php含む）
   - DB: XServerパネルのphpMyAdminでエクスポート（またはXServer自動バックアップ確認でも可）
   - ユーザーに「バックアップ取った、消していいか」を確認してから次へ
2. **btg-komaki-site リポジトリにXServer自動デプロイを追加**
   - btg-komaki-hpの `.github/workflows/deploy-xserver.yml` を流用
   - server-dir: `/sekkotsuin-komaki.com/public_html/`
   - exclude に `jiko.sekkotsuin-komaki.com/**` と `.htaccess` と旧WP関連(wp-*, blogなど)を必ず追加
   - Secrets(XSERVER_FTP_*)はbtg-komaki-hpと同じ値を登録（FTP: deploy@sekkotsuin-komaki.com）
3. **旧WPファイルの撤去**（バックアップ確認後）
   - public_html直下の wp-admin/ wp-includes/ wp-content/ wp-*.php index.php を削除
   - ※WordPress簡単インストールの管理上は残るが実体を消す（XServer APIのDELETE /wpでもよい。ただしid間違い厳禁: 2eb7a0a47f8b7f8b が旧公式。blogの073b51032a70e2b5は絶対触らない）
4. **新サイトをデプロイ** → https://sekkotsuin-komaki.com/ で表示確認（SSLは設定済みのはず。旧サイトがhttpsで動いていた）
5. **事故サイト側のリンク張り替え**
   - btg-komaki-hp内の `https://bonetechgroup.github.io/btg-komaki-site/` を `https://sekkotsuin-komaki.com/` に全ファイル一括置換
6. **旧github.io側の処理**
   - btg-komaki-siteのGitHub Pagesにカスタムドメイン設定 or 全ページmeta refreshで新URLへ転送
7. **仕上げ**
   - スマホ390px検証（iframe方式）→ プッシュ
   - Search Consoleに sekkotsuin-komaki.com のプロパティ追加＋サイトマップ送信
   - ユーザーへ: Googleビジネスプロフィールのサイト欄が旧URLならそのままでOK（ドメイン同じ）、github.ioを載せていた媒体があれば差し替え
   - メモリ（小牧院公式サイト.md）を更新
